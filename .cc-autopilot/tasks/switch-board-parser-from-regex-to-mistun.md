# Switch Board parser from regex to mistune AST + explicit emitter

## Goal

Eliminate the last regex-based markdown parser in ap2. `board.py` still
holds four regexes — `SECTION_RE`, `TASK_LINE_RE`, `_BLOCKED_CLAUSE_RE`,
`_BLOCKED_TOKEN_SPLIT_RE` — for header detection, task-line parsing, and
blocker-clause extraction. Replace the *parse* path with a mistune AST
walk; keep the *emit* path through an explicit `Task.render()` (mistune
is parse-only, and a freeform AST→markdown round-trip would lose the
canonical form).

## Context — finishes TB-102

TB-102 (commit 12d52ac) migrated `verify.py`, `config.py`, and `init.py`
off regex onto mistune. Real-world validation: 137 stoch briefings
parsed cleanly post-migration (87 with Verification sections, 50
skip-path, 0 misclassified). `mistune>=3.0` is already a hard
dependency. This task is the unfinished half of TB-102 — the Board
parser is the only remaining regex-against-agent-authored-markdown
surface.

## Driver

TB-92's case (`**TB-59** (7735de2) **Title**` injecting parenthetical
between id and title) made a task invisible to `iter_tasks()` and
silently blocked dependents until the operator noticed. The current
malformed-line event (TB-68 + TB-92) catches a subset, but only for
lines under a section heading that fail `TASK_LINE_RE` — it doesn't
cleanly catch shape-violating injections, and it's reactive not
preventive.

Acknowledge: incidents have been operator-edit-induced, not
agent-induced. This is preventive cleanup, not a fix for an active
wound. **Hold behind any task with concrete user impact.**

## Scope

- Parse: `mistune.create_markdown(renderer='ast')` walking the document
  tree. Section nodes = level-2 headings whose text is in `SECTIONS`.
  Task nodes = `list_item`s whose first inline-content node is a
  checkbox followed by a strong-emphasis run (TB-id), then another
  strong-emphasis run (title), optional codespan tags, em-dash
  separator, optional briefing link.
- Emit: keep `Task.render()` as the canonical writer. No autogenerated
  markdown.
- `board_malformed_line` events still fire for AST nodes that fail the
  task-shape check (preserve TB-68 behavior).

## Migration plan

1. Add the AST parser as a sibling to the regex one.
2. Run BOTH in parallel for a soak period and emit
   `board_parser_disagreement` when they diverge.
3. Once **stoch** (not autopilot2) reports zero disagreements over a
   24h window, drop the regex parser. autopilot2's board is too small
   to exercise enough variance — soak validation must happen on stoch.

## Out of scope

- Changing the on-disk format.
- Changing `Task` fields.
- Widening the contract to accept shapes the regex rejected. If the AST
  parser starts accepting "tags in different order" or "multi-paragraph
  description," that is a separate design decision and must be called
  out explicitly — `iter_tasks()` does not currently model multi-
  paragraph descriptions, and tag order is not semantically meaningful
  today.

## Acceptance cases — the four shapes

The original framing listed four cases. Two are concrete; two are
weaker and require justification before being accepted:

- **CONCRETE** Extra whitespace between syntactic tokens.
- **CONCRETE** Parenthetical injection between TB-id and title (the
  TB-92 case).
- **WEAKER** Tags in different order — verify whether order is or
  should be meaningful before accepting.
- **WEAKER** Multi-paragraph description — verify the Board model
  supports it before accepting; if not, this is contract widening (out
  of scope).

## Verification

The round-trip property test is the load-bearing criterion. Other
bullets are supporting evidence.

- [shell] `uv run pytest -q ap2/tests/test_board.py` (existing suite
  still green — regression gate)
- **GATING** New property test in `test_board.py`:
  `parse(render(task)) == task` for every `Task` field, across the full
  set of stoch's TASKS.md fixtures (use `tests/e2e/fixtures/` or
  equivalent).
- The AST parser accepts every shape the current regex parser accepts
  (parity test against the same fixtures).
- The AST parser additionally accepts the two CONCRETE shapes the
  regex rejects.
- During the soak window, `board_parser_disagreement` event count = 0
  on stoch over 24 contiguous hours.
